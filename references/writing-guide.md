# 小红书写作指南

## 写作原则

### 语言风格
- 口语化，像和朋友聊天
- 避免官方腔、行业黑话
- 每段最多 3 句话，句子短、有节奏感
- 大量使用 emoji 制造视觉停顿

### 结构框架
```
开头 Hook（前 3 行抓住注意力）
↓
核心内容（多段落 + 小标题 + 配图）
↓
优缺点 / Tips / 评分（增加信任感）
↓
结尾 CTA（问问题，求推荐，求点赞收藏）
```

### 互动设计
- 结尾必须有问题引导评论
- 用"姐妹们 / 宝子们"拉近距离
- 求点赞、收藏放结尾，不要放开头

### 互动数据权重（高→低）
1. 收藏（权重最高）
2. 评论
3. 点赞
4. 转发

## 排版规范

| 项目 | 设置 |
|------|------|
| 字体 | -apple-system, 'PingFang SC', 'Helvetica Neue' |
| 正文字号 | 15-16px |
| 正文颜色 | #333 |
| 标题颜色 | #fe2c55（小红书红）|
| 行高 | 1.8-2.0 |
| 段落 | 最多 3 行，段间空一行 |
| 强调文字 | `<strong style="color:#fe2c55;">重点</strong>` |

## 配图原则

- 封面图最重要，决定点击率
- 封面图比例：3:4 竖图或 1:1 方图
- 正文图片 3-9 张，**避免 2 张**
- 图片要明亮、高饱和、有生活感
- 禁止在图片上加水印、二维码、联系方式

## 避坑清单

- 标题 ≤ 20 字，禁用"最好 / 第一 / 唯一"等绝对词
- 正文 500-1000 字
- 话题标签 3-5 个，精准 > 泛用
- 不要放外部链接（会被降权）
- 发布后 30 分钟内自己评论，触发系统推荐
